## Engineering Intake Standard Operating Procedure (SOP)

### Purpose

This document defines how incoming engineering requests are evaluated, prioritized, and routed to the appropriate team.

### Scope

*This SOP applies to all incoming tasks submitted via ClickUp under the New Intake status, including:*

- Feature requests
- Bug reports
- Technical investigations
- Operational tasks

### Intake Requirements

*Each intake request should include:*

- Title: Clear and concise summary
- Description: Detailed explanation of the request
- Business Impact: Why this matters
- Requested Timeline (if applicable)
- Requester Team/Owner

If any of the above is missing, the task should be flagged as **Incomplete Intake**.

### Intake Review Process

**Step 1: Validate Submission**
Ensure required fields are filled
Clarify ambiguous or missing information
Add labels if applicable (e.g., bug, feature, ops)

**Step 2: Categorize Request**

Assign one of the following categories:

- Bug
- Feature Request
- Enhancement
- Technical Debt
- Support / Operations

**Step 3: Assess Priority**

Use the following priority levels:

**P0 (Critical)**
- Production outage
- Security issue
- Major revenue impact
**P1 (High)**
- Core functionality broken
- High customer impact
**P2 (Medium)**
- Moderate issue or improvement
**P3 (Low)**
- Minor issue or nice-to-have

**Step 4: Assign Ownership**

Ownership should be assigned based on:

Platform Team → infrastructure, APIs, backend systems
Frontend Team → UI/UX issues
Data Team → analytics, pipelines, reporting
Support Team → customer-related issues

If unclear, assign to Engineering Manager for triage.

**Step 5: Define Next Action**

Every intake must have a clear next step:

- Move to Backlog
- Move to In Progress
- Request more information
- Reject (with reason)
